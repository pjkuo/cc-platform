核心觀念：① 硬體是身體，軟體是大腦下的指令，網路是對外的神經。② 任何一個動作都是「由上往下穿過每一層」再回報——少了中間任一層，動作就會卡在那裡。③ 學會在三層之間定位問題（是硬體壞、軟體沒裝、還是網路不通），就掌握了故障排除的關鍵。

    軟體的五個子項目（由底層到上層）

    軟體不是只有「應用程式」。它由低到高分層，每一層都依賴下一層。

    層級子項目負責什麼
    ① 韌體BIOS / UEFI內建在主機板，開機時最先執行，喚醒硬體
    ② 作業系統Windows / Linux / macOS管理硬體、檔案、記憶體與所有程式的「總管」
    ③ 驅動程式顯示卡／印表機驅動…讓作業系統「聽得懂」特定硬體
    ④ 公用程式防毒／磁碟清理…維護與保護系統的工具
    ⑤ 應用程式瀏覽器／文書／遊戲使用者真正在用的軟體

    軟體安裝模擬：依正確順序安裝 → 啟動測試

    安裝佇列（順序很重要）

    軟體必須由底層往上裝。跳過下層直接裝上層會失敗。韌體已內建，請依序安裝其餘四層，再按「啟動測試」。

    縱向運作：一個指令往下穿過軟體層

    系統裝好後，按按鈕看「開啟相片」如何由應用程式逐層下達到硬體。少裝某層會卡住。

    網路的六個子項目

    要讓電腦上網，下列設定缺一不可。每一項負責通訊路上的一段。

    子項目作用少了它會怎樣
    網路卡 NIC電腦對外通訊的硬體出入口完全沒有網路
    IP 位址電腦在網路上的門牌無法被找到、無法連線
    子網路遮罩分辨「自己這區」與「外面」區網／外網判斷錯誤
    預設閘道 Gateway通往外面網路的出口（路由器）只能連區網，無法上網
    DNS 伺服器把網域名稱翻成 IP（電話簿）IP 通但打不開網站
    數據機／路由器連到 ISP，把家裡接上網際網路整個家庭斷網

    連線設定模擬：填參數 → 連線測試（ping）

    網路設定

    為這台電腦選擇設定值（有些選項是錯的，故意讓你練習找錯）。設定後按「連線測試」。

    尚未測試。設定完成後按「連線測試」。

    封包流向：你的網頁請求怎麼出去又回來

    連線成功後，按按鈕看一次「打開網站」的封包旅程。哪一段斷掉，動畫就停在那裡。

    常見網路徵狀與處理方向。

    徵狀可能原因處理建議
    完全沒有任何網路圖示網路卡停用／未安裝啟用或安裝網路卡
    IP 顯示 0.0.0.0 或 169.254.x.x沒拿到 IP（DHCP 失敗）重新取得 IP、檢查路由器

    整個家裡都斷網數據機／ISP 線路問題重開數據機、聯絡 ISP
    網路很慢、常斷線訊號弱／頻寬被占用靠近路由器、檢查占用
