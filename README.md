# 計算機概論模擬平台

一個**單一 HTML 檔、零後端、可離線使用**的計算機概論教學與評量平台，專為一年級（含高職生銜接）設計。以「硬體 → 軟體 → 網路」三大主軸為骨架，結合互動模擬、內建教學講義、即時評量與具名課後評量，並採「**先思考後驗證**」的教學理念。

- 檔案：`計算機概論模擬平台.html`
- 技術：純 HTML / CSS / JavaScript，**無需安裝、無需伺服器、無外部相依套件**
- 使用方式：用瀏覽器（Chrome／Edge／Firefox／Safari）直接開啟即可

## 快速開始

1. 下載 `計算機概論模擬平台.html`。
2. 用瀏覽器**雙擊開啟**（或拖入瀏覽器視窗）。
3. 首頁即為「課程架構與導覽」，點各模組的「前往」按鈕進入子頁。

> 完全離線可用；若要全班共用課後評量成績統計，建議部署到網站（見[部署方式](#部署方式)）或於同一台機器操作。

## 頁面架構

平台採**分層式導覽**，首頁為架構中心，子頁皆可一鍵返回首頁。

```
首頁（課程架構與導覽）
├── 學習模組
│   ├── 硬體 Hardware
│   ├── 軟體 Software
│   └── 網路 Network
│       每個單元頁皆含「學習路徑」三層：
│       ① 互動模擬  →  ② 教學講義  →  ③ 課後評量
└── 評量系統
    ├── 即時評量（課堂・形成性）
    └── 課後評量系統（課後・總結性）
        ├── 學生測驗
        └── 成績統計（教師，需密碼）
```

首頁另含「**橫向資料流：訊息封包雙向傳送**」動畫：示範一個「存檔」動作如何由應用層向右逐層傳到硬體與雲端（請求），再向左逐層回報（回應）。

## 功能總覽

| 模組 | 內容 |
|------|------|
| **三大主軸互動模擬** | 硬體（安裝・開機 POST・故障診斷）、軟體（分層安裝・運作・除錯）、網路（IP/DNS/閘道設定・封包流向・連線測試） |
| **內建教學講義** | 每單元約 15–20 張、由單元內容自動生成的投影片，可翻頁與全螢幕簡報；重點以**紅字**標示，附**參考書籍與連結** |
| **即時評量** | 課堂用混合出題；先思考後驗證、把握度校準、運算思維（CT）四構面分析、錯題回顧、CSV 匯出 |
| **課後評量系統** | 具名測驗（姓名／學號／班級）、約 **30 分鐘倒數計時**、自動交卷、複習錯題；成績自動記錄 |
| **成績統計（教師）** | 密碼保護；次數／人數／平均／及格率、分數分布、主軸與 CT 平均答對率、作答紀錄表、CSV 匯出 |
| **學習進度追蹤** | 每單元「學習路徑」三層完成後顯示 ✓（模擬、讀完講義、完成課後評量），提升學生參與度 |

## 學生使用流程

每個單元頁頂端的「**學習路徑**」即建議流程：

1. **① 互動模擬** — 動手操作該單元的模擬（安裝、開機、設定、排錯），先預測再驗證。
2. **② 教學講義** — 翻閱內建投影片或點「⛶ 全螢幕簡報」聽講；紅字為重點與課後評量考點。
3. **③ 課後評量** — 進入具名測驗（約 30 分鐘），作答後可查看解析與**複習錯題**。

完成各層後，對應的步驟標籤會變綠並打勾（✓），進度會記在該瀏覽器中。

## 教師使用說明

### 進入成績統計

1. 首頁 →「前往課後評量」→ 切換到「**成績統計（教師）**」分頁。
2. 輸入教師密碼後按「進入」。
   - **預設密碼：`npust1728`**
   - 解鎖狀態維持到該瀏覽器分頁關閉；重新整理需再次輸入。

### 統計內容

- **概覽卡**：測驗次數、學生人數、平均分、最高/最低分、及格率
- **分數分布**長條圖（90–100 / 80–89 / 70–79 / 60–69 / 0–59）
- **主軸與 CT 四構面**平均答對率
- **作答紀錄表**：時間、姓名、學號、班級、範圍、難度、題數、分數、等第、用時
- **匯出全部成績（CSV）**、**清除所有紀錄**

### 課後評量設定（學生測驗分頁）

- 範圍（綜合／硬體／軟體／網路）、難度（全部／基礎／進階）、題數（5／10／全部）
- 「複習模式」為不計分練習，不納入統計

## 內建教學講義

- 每單元投影片**取材自該單元內容、依架構分層編排**，並**與課後評量題目高度對應**：
  - **硬體**：五大單元 → 核心元件 → 開機 POST → 關機程序 → 故障診斷 → 動手做
  - **軟體**：軟體分層 → 開機順序 → 作業系統角色 → 驅動程式 → IPO/迴圈/函式 → 故障排除 → 動手做
  - **網路**：連線要素 → 資料路徑 → DNS 解析 → IoT/行動網路/MQTT → 故障定位 → 連線測試 → 動手做
- 操作：`◀ 上一張` / `下一張 ▶`、頁碼、`⛶ 全螢幕簡報`（深色大字，支援鍵盤 **← →**、空白鍵前進、**Esc** 關閉）。
- 重點以**紅色強調**；每單元最後一張為「參考資料」，列出書籍與連結。
- **外部講義**：點「＋ 外部講義」展開，可貼上 Google 簡報／PDF／公開 PPT 連結，或開啟本機 PDF（連結會記住）。
  - PPT（`.pptx`）公開連結會自動以 Microsoft Office Online 檢視器內嵌。
  - Google 簡報建議使用「發布到網路／embed」或 `/preview` 連結（一般 `/edit` 連結可能被禁止內嵌）。
  - 本機 PPT 請先轉成 PDF。

## 資料儲存與隱私

- 所有資料（課後評量成績、學習進度、外部講義連結）皆存於**使用者瀏覽器的 localStorage**，**不會上傳到任何伺服器**。
- 因此資料是**「綁定該瀏覽器／該機器」**：
  - 全班共用統計時，建議學生在**同一台機器**作答，或各自**匯出 CSV** 後由教師彙整。
  - 清除瀏覽器資料、換機器或無痕模式會看不到先前紀錄。
- localStorage 使用的鍵：`cc_exam_records_v1`（成績）、`cc_progress_v1`（學習進度）、`cc_slides_<unit>`（外部講義連結）。

## 部署方式

擇一即可：

- **離線單機**：直接用瀏覽器開啟 HTML 檔（最簡單）。
- **校內/雲端網站**：將 HTML 檔上傳到任何靜態主機即可，例如：

## 自訂與維護

平台為單一檔案，可用文字編輯器開啟修改：

- **修改教師密碼**：搜尋 `TEACHER_PW="npust1728"`，改成你要的密碼。
- **增刪題庫**：搜尋 `CC_BANK`（即時評量與課後評量共用），每題格式：
  ```js
  {cat:"hw", ct:"de", lv:1, stem:"題目…", opts:["A","B","C","D"], ans:1, exp:"解析…"}
  ```
  - `cat`：主軸 `hw`/`sw`/`nw`；`ct`：CT 構面 `de`(拆解)/`pa`(模式)/`ab`(抽象)/`al`(演算)；`lv`：難度 `1`(基礎)/`2`(進階)；`ans`：正解索引（從 0 起算）。
- **修改講義內容**：搜尋 `var DATA=`（內建講義），每張投影片格式：
  ```js
  {k:"小標", t:"標題（用 *星號* 包住會變紅字）", pts:["條列…（同樣可用 *…* 標紅）"], ref:[{label:"書名"},{label:"連結文字",url:"https://…"}]}
  ```
- **調整作答時間**：搜尋 `EXAM_SEC=30*60`（單位為秒）。

> 修改後務必用瀏覽器重新整理測試。建議改動前先備份原檔。

## 瀏覽器需求與已知限制

- 建議使用最新版 **Chrome／Edge／Firefox／Safari**。
- 互動模擬、講義翻頁、計時、動畫等皆以標準 JavaScript 實作，需在**一般瀏覽器**中執行（部分老舊的離線截圖／預覽引擎不會執行 JavaScript）。
- 外部講義內嵌受對方網站的內嵌政策影響；若被禁止內嵌，可用「開新視窗」改在新分頁開啟。
- 成績統計為**本機端**統計，非跨裝置雲端同步（如需跨裝置彙整，請用 CSV 匯出）。

## 教學理念對應

- **先思考後驗證**：學生先作答／預測，再由系統或 AI 驗證與回饋。
- **把握度校準**：作答時標註把握度，結束後檢視「答對且有把握／答錯卻有把握（迷思）」等狀態。
- **運算思維（CT）四構面**：題目標註拆解／模式辨識／抽象化／演算法，並於成績中呈現各構面表現。
- **分層學習路徑**：互動模擬 → 教學講義 → 課後評量，逐層完成、可視化進度。
